이 웹앱의 UI 스타일만 “20,30대 여성이 좋아할 만한 예쁘고 귀여운 감성”으로 리디자인해줘.

절대 조건:
1. 기능, 로직, 데이터 구조, 상태 관리, 오디오 처리, 단축키, 저장/불러오기, GitHub Pages 설정은 절대 수정하지 마.
2. `src/store`, `src/utils`, `src/audio`, `src/types`의 동작 코드는 건드리지 마.
3. TS/TSX 파일은 className 추가나 스타일 연결이 정말 필요한 경우에만 최소 수정하고, 이벤트 핸들러/함수/상태 로직은 절대 바꾸지 마.
4. 주요 수정 대상은 CSS 파일과 디자인 토큰이어야 한다.
5. UI 텍스트, 메뉴 구조, 버튼 기능, 단축키 매핑은 그대로 유지해.
6. 작업 후 `git diff`를 확인해서 기능 코드 변경이 섞였으면 즉시 되돌려.

스타일 목표:
- “귀엽지만 유치하지 않은”, “작업용 앱답게 정돈된”, “부드럽고 사랑스러운 음악 작업실” 느낌.
- 20,30대 여성 사용자가 좋아할 만한 감성적이고 깔끔한 UI.
- 너무 진한 네온, 과한 그라디언트, 장식용 blob/orb, 과한 그림자, 마케팅 랜딩페이지 같은 느낌은 피한다.
- 피아노롤/작업 영역의 가독성은 최우선으로 유지한다.
- 노트, 그리드, 선택 상태, 재생 헤드, velocity lane은 기존보다 보기 편해야 한다.

추천 비주얼 방향:
- 배경: 아주 어두운 블랙 대신 부드러운 딥 차콜 또는 따뜻한 아이보리 라이트 모드 중 현재 앱 구조에 더 잘 맞는 방향 선택.
- 포인트 컬러: soft rose, coral pink, fresh mint, butter yellow, lavender-gray를 균형 있게 사용.
- 단일 핑크톤으로만 밀지 말고, 음악 작업 앱답게 contrast를 확보한다.
- 버튼: 작고 귀여운 pill 느낌은 허용하되, 텍스트가 잘리지 않게 하고 hover/active/focus 상태를 명확히 만든다.
- 패널: 8px 이하의 적당한 radius, 얇은 border, 은은한 shadow.
- 툴바: 아이콘 버튼이 더 사랑스럽고 명확하게 보이도록 active 상태를 개선.
- 입력/셀렉트/슬라이더: 작고 정돈된 컨트롤로 통일.
- 피아노롤 캔버스 주변: grid와 패널 경계가 부드럽지만 작업 구분은 명확하게.

수정 후보 파일:
- `src/styles/global.css`
- `src/components/layout/AppShell.css`
- `src/components/pianoRoll/ToolBar.css`
- `src/components/pianoRoll/PianoRoll.css`
- `src/components/transport/TransportBar.css`
- `src/components/panels/InspectorPanel.css`
- `src/components/panels/TrackPanel.css`
- 기타 CSS 파일만 필요 시 수정

구현 단계:
1. 먼저 전체 CSS 변수/디자인 토큰을 파악해.
2. 색상 팔레트를 새로 정의하되 contrast를 확인해.
3. 툴바, 패널, 트랙 리스트, 인스펙터, 트랜스포트, 피아노롤 주변 UI 순서로 스타일만 개선해.
4. 캔버스 내부 로직은 건드리지 말고, 캔버스 배경/노트 색은 기존 prop이나 CSS 변수로 가능한 범위에서만 조정해.
5. 모든 텍스트가 버튼/패널 밖으로 넘치지 않게 확인해.
6. 모바일/작은 화면에서도 toolbar가 깨지지 않게 wrap/overflow를 정리해.
7. `npm run build`를 실행해 타입/빌드가 깨지지 않는지 확인해.
8. 가능하면 Playwright로 화면을 열어 빈 화면, 콘솔 에러, UI 겹침이 없는지 확인해.

완료 기준:
- 기능 변경 diff가 없어야 한다.
- UI만 더 예쁘고 귀엽고 부드러운 분위기로 바뀌어야 한다.
- 버튼, 슬라이더, 패널, 툴바, 피아노롤 주변 레이아웃이 깨지지 않아야 한다.
- `npm run build` 통과.
- 최종 응답에는 “수정한 CSS 파일 목록”과 “기능 변경 없음 확인”을 반드시 포함해.
